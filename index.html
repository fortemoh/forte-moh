New CVARs

These CVARs did not exist before and were introduced in the ForteAC builds. Set them in the console or in your server config.

g_roundstartdelay
Default: 5 ARCHIVE

Freeze duration (in seconds) at the start of each round for round-based game types. Players cannot move during the countdown. A message sequence counts down from this value and ends with "Go, GLHF!". Set to 0 to disable the freeze entirely.

Usage: set g_roundstartdelay 5

g_preventghostwalk
Default: 0 ARCHIVE

Prevents "ghost walking" — an exploit where rapidly switching weapons resets leg animation timing, making the player appear to glide. When enabled (1), leg animation timing is preserved across weapon switches.

Usage: set g_preventghostwalk 1

g_headstand
Default: 0 ARCHIVE

Controls whether players can stand on other players' heads (headstand / head-pushing). When set to 0 (default), head-standing is disabled, restoring classic 1.x behavior. Set to 1 to allow players to jump on and stand on each other.

Usage: set g_headstand 0

g_spectatefix
Default: 1

Enables the spectator system fix. When active, corrects issues with spectator mode behavior. Set to 0 to revert to original spectator behavior.

Usage: set g_spectatefix 1

g_spectatefollow_forward
Default: 0

Controls the forward/back offset of the spectator follow camera. Previously hardcoded to -56. Now defaults to 0 so the camera sits directly behind the followed player. Negative values move the camera further back.

Usage: set g_spectatefollow_forward 0

g_spectatefollow_up
Default: 0

Controls the vertical (up/down) offset of the spectator follow camera. Previously hardcoded to 24. Now defaults to 0 for an eye-level perspective. Positive values raise the camera.

Usage: set g_spectatefollow_up 0

sv_autokick
Default: 0 ARCHIVE

Automatically kicks clients that fail sv_pure file validation after the specified number of seconds. Requires sv_pure to be enabled. When a client is detected as "unpure" (invalid or missing pk3 files), a timer starts and the client is kicked when it expires. Set to 0 to disable auto-kicking.

Usage: set sv_autokick 30 — kick unpure clients after 30 seconds

s_show_num_active_sounds
Default: 0 CHEAT

Debug CVAR. When enabled, displays the number of currently active sounds. Requires cheats to be enabled (sv_cheats 1).

Usage: set s_show_num_active_sounds 1

cg_showemitters
Default: 0 CHEAT

Debug CVAR. When enabled, renders the locations of particle emitters in the world. Requires cheats to be enabled.

Usage: set cg_showemitters 1

com_target_game
Default: (build-time) INIT | PROTECTED

Allows a build-time override of the default target game via the DEFAULT_TARGET_GAME compile macro. Cannot be changed at runtime (CVAR_INIT). Protected from VM modification.

All Changes

Anticheat & sv_pure

  • New on-screen Anticheat HUD (configstring index 255) in bottom-right corner shows real-time sv_pure status: green "Clean (X/Y)", red "Unclean (X/Y)", or grey "Anticheat Off"
  • sv_autokick system: auto-kicks unpure clients after a configurable delay
  • Broadcasts names of unclean players to all clients at round start
  • Dozens of renderer and client CVARs locked behind CVAR_CHEAT to prevent visual exploits

Gameplay

  • Round-start freeze: configurable countdown via g_roundstartdelay with on-screen messages
  • Ghost walk prevention: g_preventghostwalk preserves leg animation across weapon switches
  • Head-stand control: g_headstand disables standing on players' heads (restores 1.x behavior)
  • Fall damage tweaked (SH/BT protocol): reduced by 1 point at each severity level
  • Damage angle fix: corrected bug where damage yaw was clamped instead of roll, fixing spurious view tilt
  • FOV range tightened: 80–96.42 (was 65–120) for competitive fairness

Spectator

  • Follow-camera offsets reset to 0 (was -56 forward, 24 up) for direct behind-player view
  • Spectator fix enabled by default via g_spectatefix
  • Removed hint text overlay ("Press Use to follow player", etc.) from spectator view

Networking & Performance

  • com_maxfps raised from 85 to 250
  • cl_maxpackets raised from 30 to 125 (range-clamped)
  • cl_timeNudge clamped to [0, 20]
  • cl_pitchspeed halved from 140 to 70

Display & Rendering

  • Resolution presets updated: modes 0/1/10/11 now map to 720p/1080p/1440p/4K
  • r_gamma maximum extended from 1.0 to 2.0
  • r_znear reduced from 4 to 3; r_ambientScale from 0.6 to 0.5

UI & Client

  • Console enabled by default (ui_console 1)
  • Automatic demo recording on by default (cl_autoRecordDemo 1)
  • Intro sequence skipped (cl_playintro 0 forced)
  • Ambient sounds disabled by default (s_ambientVolume 0)
  • Server lag display forced off and read-only (cg_drawsvlag 0, ROM)
  • Team kill kick disabled by default (g_teamkillkick 0)

Launchers

  • macOS: New native SwiftUI launcher with game launching and settings persistence
  • Windows: GUI launcher rewrite with bookmark support, icon display, and Discord button

Build System

  • CI artifact naming switched to role-first format (removed "out-" prefix)
  • New branding icons across platforms
  • Windows builds use static CRT (eliminates MSVCP140.dll dependency)
